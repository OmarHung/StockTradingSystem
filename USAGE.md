# 使用手冊 — 台股智慧交易終端

從零開始到每天無人值守自動交易的完整操作指南。
（系統架構與開發規劃見 [`README.md`](README.md) / [`PLAN.md`](PLAN.md)）

---

## 1. 安裝與啟動

```bash
# 首次安裝（⚠️ Apple Silicon 一律加 arch -arm64，否則裝到 x86_64 版會壞）
python3 -m venv .venv
arch -arm64 .venv/bin/pip install -r requirements.txt
cd frontend && npm install && cd ..

# 啟動（每次使用都執行這個；後端 :8000 + 前端 :5173）
bash scripts/dev.sh
```

瀏覽器開 **http://localhost:5173**。

> ⚠️ **排程依賴後端常駐**：內建排程器住在後端行程裡，`dev.sh` 關掉排程就停。
> 要無人值守，讓這個終端機視窗一直開著（或把後端掛成常駐服務）。

## 2. 初次設定（只做一次）

### 2.1 填 API 金鑰（⚙️ 設定 → 🔑 API 金鑰）

| 金鑰 | 用途 | 必要性 | 取得 |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | AI 分析/決策/反思 | AI 功能必填 | console.anthropic.com |
| `SJ_API_KEY` + `SJ_SEC_KEY` | 股價主源、排行榜 | 強烈建議 | sinotrade.com.tw → Python API |
| `FINMIND_TOKEN` | 指數價格 + 備援 | 建議 | finmindtrade.com 免費註冊 |

金鑰寫入 `.env`（不進版控）。同頁下方可切換券商 **模擬 ↔ 正式** 環境（Phase 6 前請保持模擬）。

### 2.2 首次資料回補（📦 資料）

打開資料視窗 → 保持預設（全市場、全部資料類型勾選、額度自動等待開）→ 按「**回補**」。

- 首次全市場約 2 年資料需數小時（可過夜跑；隨時可「中止」，下次自動續補不重抓）
- 進度列會顯示目前在補哪一天/哪一檔；撞到 FinMind 額度會顯示「等待額度」並自動續跑
- 跑完按「品質檢查」確認無真實缺日

### 2.3 檢查資料健康

資料視窗頂部的健康報告應該全綠（覆蓋率 85%+ 屬正常——分母含已下市股票；
除權息 ~79% 是自然上限，沒配息的公司本來就沒有除權息資料）。

## 3. 介面導覽

### 頂部工具列
- **大盤指數**（加權/櫃買）、時鐘、資料連線狀態、AI 啟用徽章、券商環境徽章
- **🧪 回測**：策略回測獨立視窗
- **🗂 股票**：全市場瀏覽器
- **📦 資料**：健康報告 + 回補 + 品質檢查
- **⚙️ 設定**：資金/風險/選股/LLM/排程/金鑰

### 主畫面面板（可拖曳標題列調整佈局）
| 面板 | 用途 |
|---|---|
| ⭐ 自選清單 | 點列切換主圖；星號加入/移除 |
| 🏆 排行榜 | 漲跌幅/成交值排行（shioaji 即時） |
| 📈 K 線圖 | 日/週/月切換；**還原↔原始價**切換 |
| 🔍 智慧選股 | 多因子量化初篩 Top30 + 因子拆解 |
| 🧑‍💼 AI 選股報告 | 分析師團隊 → 交易員計畫 → 風控核准 |
| 💰 持倉績效 | 權益曲線 vs 大盤、持倉/成交、每日流程、緊急停止 |
| 🧠 大腦活動 | 每次 LLM 呼叫的 prompt/回應/驗證攔截 |
| 📚 反思規則庫 | AI 歸納的規則/反模式，可停用、一鍵反思 |

### 🗂 股票瀏覽器
- 列表含**開高低收/漲跌幅**、當月除權息黃標（含金額）、處置標記
- 篩選：市場/產業/狀態（含「當月除權息」）；搜尋代號或名稱
- **點股票 → 整頁詳情**，五個分頁：
  - **總覽**：走勢圖 + 關鍵數據摘要
  - **籌碼**：外資/投信買賣超圖、融資餘額圖、法人五日淨買
  - **基本面**：月營收圖、本益比走勢、PER/PBR/殖利率、本益比一年位階
  - **除權息**：未來預告（日期+金額）+ 歷史明細（含每次調整幅度）
  - **資料覆蓋**：這檔股票各資料集的起迄與列數

## 4. 日常操作流程

### 全自動模式（建議）

什麼都不用做。內建排程器平日自動執行：

| 時間 | 任務 | 內容 |
|---|---|---|
| 09:00 | 盤中停損監控 | 每 30 秒輪詢持倉快照，盤中觸及停損/停利立即出場（收盤/無持倉自行結束） |
| 14:30 | 資料增量更新 | 補當日股價/籌碼/估值/除權息/營收 |
| 15:00 | 每日交易主流程 | 撮合昨日委託 → 停損停利(補收盤檢查) → 權益快照 → (週五)反思 → AI 決策 → 風控 → 掛明日單 |

監控與調整在 **⚙️ 設定 → ⏰ 排程**：啟停開關、時間、上次/下次執行、log、▶ 立即執行。

### 手動模式

1. **🔍 智慧選股** 按「選股」→ 看 Top30 與因子分數
2. **🧑‍💼 AI 選股報告** 按「產生報告」→ 三位分析師 + 交易員給出進場區間/停損/目標價，
   風控閘門顯示 🛡️ 核准股數或駁回原因
3. **💰 持倉績效** 按「▶ 執行每日流程」→ 依計畫掛單（隔日開盤有效）
4. 隔天看持倉面板的成交與損益

### 檢視 AI 的思考

- **🧠 大腦活動**：每次呼叫的完整 prompt/回應；「驗證攔截」標記 = AI 引用的數字與實算不符被降信心
- **📚 反思規則庫**：每週五自動反思產出的規則與反模式（如「避免在超買時追高」），
  會自動注入之後的決策 prompt；覺得某條規則有害可以停用

## 5. 策略回測（🧪 回測）

1. 選策略：**多因子選股** / **多因子+風控**（ATR停損+冷卻+大盤濾網）/ 買進持有 0050 / 0050 均線
2. 設區間（預設近 2 年）、初始資金、持倉上限 → 按「回測」
3. 執行中顯示實時 log；完成後顯示總報酬/年化/Sharpe/MDD/交易數 + 權益曲線 + 逐筆明細
4. 關閉視窗不會重置——回測跑一半關掉，重開繼續看

> 回測一律用**還原價**（除權息+分割+減資已調整），與含息報酬一致。

## 6. 風險控管（自動生效，無需操作）

AI 的任何買進計畫都必須通過九道規則閘門（LLM 不可逾越）：

處置股黑名單 → 回撤熔斷(DD≥15%停新倉) → 計畫合理性 → R:R≥1.5 → 停損冷卻(5日)
→ 部位規模(單筆風險≤資金1%) → 單股上限15% → 產業曝險30% → 現金/持倉數上限

參數在 **⚙️ 設定 → 🛡️ 風險** 調整。被駁回的交易記錄在 friction log（AI 報告卡上可見原因），
且會被反思引擎拿去檢討「如果當初成交會如何」。

### 緊急停止
**💰 持倉績效 → 🛑 緊急停止**：立即停止開新倉，只做保護性出場（停損/停利照常）。
再按一次恢復。♻️ 重置帳本 = 清空模擬帳戶重新開始（正式驗收前用）。

## 7. 常見問題

| 狀況 | 說明 / 處理 |
|---|---|
| 回補時顯示「等待額度」 | FinMind 每小時重置額度，系統自動睡到整點續跑，不用管 |
| 覆蓋率不到 100% | 分母含 ~200 檔已下市股票；活股覆蓋接近 100%，正常 |
| 除權息只有 79% | 自然上限（兩年內沒配息的公司本來就沒資料），非缺失 |
| 某股票沒 K 線 | 🗂 股票 → 該股「資料覆蓋」分頁確認；未下載的股票不在股票池（ETF/下市/處置） |
| AI 報告失敗 | ⚙️ 設定確認 ANTHROPIC_API_KEY；🧠 大腦活動看錯誤；模型故障會自動 fallback |
| 排程沒跑 | ⚙️→⏰ 看「上次執行」；後端（dev.sh）必須在跑；晚開機會自動補跑當天 |
| 回測說無價格資料 | 區間太早（資料從回補起點開始）或該市場未回補 |
| pip 裝出 x86_64 錯誤 | 重裝一律 `arch -arm64 .venv/bin/pip install ...` |

## 8. CLI 對照（全部功能 WebUI 都有，CLI 供除錯）

```bash
.venv/bin/python -m scripts.backfill --auto-wait              # 全市場回補
.venv/bin/python -m scripts.backfill --datasets valuation     # 只補某類資料
.venv/bin/python -m scripts.backfill --stocks 2330 --force    # 強制重抓單檔
.venv/bin/python -m scripts.run_daily                         # 每日交易主流程
.venv/bin/python -m scripts.run_backtest --strategy screener_risk --start 2024-07-01 --end 2026-07-04
.venv/bin/python -m pytest tests/ -q                          # 測試（45）
```

資料庫：`data/market.db`（SQLite）；日誌：`logs/system.log`、`logs/jobs/*.log`；
向量記憶：`data/chroma/`。這三個目錄都不進版控。
